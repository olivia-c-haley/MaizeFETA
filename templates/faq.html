Answers to common questions about FETA's methods, inputs, statistics, and outputs.

General

FETA (Functional Enrichment Term Analysis) is a web-based tool for performing gene set enrichment analysis in maize (Zea mays). Given a list of gene models, FETA tests whether specific biological functions, pathways, protein domains, or subcellular localizations are statistically over- or under-represented relative to a defined background population.

FETA supports both the B73 reference genome and 25 additional NAM (Nested Association Mapping) founder lines, making it broadly useful across the maize research community.

The B73 Enrichment page supports the B73 v5 reference genome. The Non-B73 Enrichment page supports 25 NAM founder lines including B97, CML52, CML69, CML103, CML228, CML247, CML277, CML322, CML333, HP301, Il14H, Ki3, Ki11, Ky21, M37W, M162W, Mo18W, Ms71, NC350, NC358, Oh7B, Oh43, P39, Tx303, and Tzi8.

FETA automatically detects which background to use based on the gene model ID prefix in your input list (e.g., Zm00001eb for B73, Zm00018ab for B97).

A manuscript describing FETA is currently in preparation. In the meantime, please cite this tool by its URL and acknowledge the annotation databases used in your analysis (see the Data Sources page for individual citation recommendations).

Input & Setup

Paste one gene model ID per line in the text box. Gene models should follow standard maize nomenclature, for example:

  • B73: Zm00001eb428070
  • B97: Zm00018ab012345

Trailing whitespace, carriage returns, and blank lines are automatically stripped. Gene model suffixes (e.g. .1, .2) are not required and are not expected.

The background (also called the universe or reference set) defines the pool of genes against which enrichment is tested. By default, FETA uses all annotated gene models for the detected genotype.

You can supply a custom background by pasting a second gene list into the background field. This is recommended when your query genes were derived from a pre-filtered experiment (e.g., only expressed genes, only genes with a protein structure prediction) — using the full genome as a background in such cases can inflate significance.

💡 Best practice: The background should represent the set of genes that could have been detected in your experiment, not all genes in the genome.

There is no hard minimum, but enrichment analyses generally perform better with larger gene sets. As a rough guide:

  • < 20 genes: Results may be unreliable due to low statistical power. Interpret with caution.
  • 20–100 genes: Reasonable for focused analyses; biological context should guide interpretation.
  • > 100 genes: Sufficient for robust enrichment detection across most annotation categories.

Very large gene lists (> 5,000 genes) may also be uninformative, as they often capture a large fraction of the genome and many terms will appear significant by chance.

Genes not present in the annotation database for the detected genotype are silently excluded from the enrichment calculation. The Load Genes button above the textarea will flag any such genes before you run the full analysis, so you can verify your list and correct any ID formatting issues in advance.

Statistics

FETA uses a one-tailed Fisher's Exact Test for each annotation term, comparing the proportion of query genes annotated to a term against the proportion in the background. The 2×2 contingency table is:

  • In term & in query
  • In term & not in query
  • Not in term & in query
  • Not in term & not in query

Each term is tested independently, then p-values are corrected for multiple comparisons.

FETA offers two correction methods, selectable in the settings panel:

  • Benjamini-Hochberg (BH / FDR): Controls the false discovery rate. Recommended for exploratory analyses where some false positives are tolerable. This is the default.
  • Bonferroni: Controls the family-wise error rate. More conservative; appropriate when you need to minimise false positives, but will reduce power considerably for large annotation sets.

The threshold you set applies to the adjusted p-value (after multiple testing correction). Only terms passing this threshold are displayed in the results. The raw p-values are retained internally and included in downloaded TSV files.

Over-enriched terms appear in your query genes at a higher proportion than in the background — suggesting your gene set is functionally biased toward that term. Under-enriched terms appear less frequently than expected, indicating your gene set may be depleted for that function.

Both directions are biologically meaningful. For example, a set of stress-responsive genes might be over-enriched for defense pathways and under-enriched for housekeeping processes like ribosome biogenesis.

Annotation Sources

FETA provides multiple GO annotation pipelines for B73, which differ in the method used to assign GO terms:

  • GOMAP: Gene Ontology Meta Annotator for Plants — a pipeline specifically designed for plant genomes that integrates multiple evidence sources.
  • PANNZER: Probabilistic ANNotation with Z-score EnRichment — a homology-based annotator that predicts GO terms from sequence similarity.
  • EnTAP: Eukaryotic Non-Model Transcriptome Annotation Pipeline — combines homology search with functional database queries.
  • GO Expanded: A combined, non-redundant set integrating all of the above with propagation up the GO hierarchy.

For most analyses, GO Expanded provides the broadest coverage. Individual pipelines are provided for transparency and comparison.

DeepLoc is a deep learning-based tool that predicts the subcellular localization of proteins from their amino acid sequences. FETA uses DeepLoc predictions to test whether your gene set is enriched for proteins targeting specific compartments such as the plastid, mitochondrion, nucleus, endoplasmic reticulum, and others.

Results are displayed as an interactive plant cell diagram where each compartment is coloured by enrichment direction and significance. Hovering over a compartment shows the −log₁₀(p) value and number of hit genes.

PhylostratrR assigns each gene to a phylostratum — a taxonomic age class representing when that gene's family first appeared in evolution. The analysis tests whether your query set is biased toward older (conserved) or younger (lineage-specific) genes. An overall Chi-squared test assesses the full distribution, while per-stratum Fisher's Exact Tests (Benjamini-Hochberg corrected) identify which specific ages are over- or under-represented.

PSAURON scores how well each gene's expression profile matches a canonical "housekeeping" pattern. FETA compares your query set's PSAURON score distribution against the genome-wide background using a Mann-Whitney U test (shift in central tendency), a Kolmogorov-Smirnov test (distributional shape), and a Chi-squared test on the binary housekeeping classification. This can reveal whether your gene set is unusually constitutively expressed or unusually condition-specific.

Annotations are tied to specific genome releases and pipeline versions and are not updated automatically. The B73 annotations are based on the B73 v5 genome assembly. Version details for each annotation pipeline are provided on the Data Sources page. If you require the most current annotations, we recommend checking the primary databases (MaizeGDB, KEGG, UniProt) directly.

RNA-seq Analysis

The RNA-seq Analysis page lets you visualise differential expression results as an interactive volcano plot and extract gene lists directly from it for downstream enrichment analysis in FETA. You can either upload your own differential expression output file or load a curated dataset from MaizeGDB.

Once a dataset is loaded, you can adjust logFC and significance thresholds by typing values or dragging the threshold lines on the plot, select up-regulated, down-regulated, or all DEGs with one click, and send the resulting gene list straight to the B73 or Non-B73 Enrichment pages without any copy-pasting.

The uploader accepts .txt, .tsv, and .csv files containing tabular differential expression output — for example, from EdgeR, DESeq2, or similar tools. The file must have a header row. After uploading, a column mapper will appear so you can assign each column to its role (gene ID, logFC, p-value, FDR, etc.).

Required columns are:

  • Gene ID — maize gene model identifiers (e.g. Zm00001eb428070). Auto-detected as the first non-numeric column.
  • logFC — log₂ fold change values.
  • FDR or p-value — choose one; if values are not already on a −log₁₀ scale, check the "already log-transformed" box accordingly.

Additional columns (logCPM, F-statistic) are optional and will be included in downloaded gene tables if present.

The MaizeGDB curated datasets are a collection of published maize RNA-seq differential expression results that have been uniformly processed and mapped to B73 v5 gene models, curated by Hayford et al. (2024). They span a wide range of tissues, developmental stages, and stress conditions.

To use them, switch to the MaizeGDB Curated Datasets mode, then search or filter studies by genotype, tissue, condition, or curation set. Click a study card to select it and press Load Selected Study to generate the volcano plot immediately — no file upload required.

💡 Because all curated datasets are mapped to B73 v5, the Send to B73 Enrichment button will always be available after loading one. The Non-B73 button will be greyed out.

Two thresholds control which genes are classified as differentially expressed:

  • logFC cutoff (±): The minimum absolute log₂ fold change. Genes with |logFC| ≥ this value are considered to have a biologically meaningful expression change. The default is ±2 (i.e. a 4-fold change).
  • −log₁₀(FDR/p-val) cutoff: The minimum statistical significance on the y-axis. A value of 2 corresponds to FDR ≤ 0.01; a value of 1.3 corresponds to FDR ≤ 0.05. The default is 2.

You can change these values by typing directly into the threshold boxes and clicking Apply, or by dragging the dashed lines on the volcano plot itself. Genes in the upper-right quadrant (high positive logFC, high significance) are up-regulated; genes in the upper-left quadrant are down-regulated.

Use the quadrant selector buttons below the volcano plot — ▲ Up-regulated, ▼ Down-regulated, or All DEGs — to choose which genes to extract. The selected genes are highlighted on the plot and listed in the gene table below it.

Once genes are selected, the Send to B73 Enrichment or Send to Non-B73 Enrichment buttons in the sidebar become active. Clicking one will open the appropriate enrichment page in a new tab with your gene list pre-filled, ready to run.

💡 The send buttons are automatically disabled if the detected genome does not match the enrichment page — for example, if your file contains Non-B73 gene IDs, the B73 Enrichment button will be greyed out.

Several download options are available from the volcano plot panel:

  • ⬇ SVG: Exports the volcano plot as a scalable vector graphic, suitable for publication figures.
  • ⬇ PNG: Exports a raster (2× resolution) image of the current plot.
  • ⬇ DEGs TSV: Downloads all genes meeting the current logFC and significance thresholds as a tab-separated file with columns for gene model, logFC, logCPM, F-statistic, p-value, FDR, and direction.
  • ⬇ Download TSV (gene table): Downloads only the genes in the currently selected quadrant.

When you send DEGs to the enrichment pages, FETA pre-fills only the query gene list. On the enrichment page, the default background is the full annotated genome for the detected genotype.

However, for RNA-seq data it is generally more appropriate to use all expressed genes (i.e. all genes tested for differential expression) as the background, not the full genome. You can do this by pasting that broader gene list into the background field on the enrichment page before running. This avoids inflated significance caused by testing against genes that were never expressed in your experiment.

💡 Best practice: Use the full list of genes with detectable expression in your RNA-seq experiment as the custom background, rather than the default whole-genome background.

Interpreting Results

The bar chart displays significant terms ranked by −log₁₀(adjusted p-value). Longer bars indicate stronger statistical evidence. Green bars represent over-enriched terms; orange/red bars represent under-enriched terms. The chart is capped at the top 30 terms by default for readability.

Note that statistical significance does not always equal biological importance — a highly significant term may simply reflect the redundancy of a large gene family. Always cross-reference results with biological knowledge of your system.

The CNET plot is a bipartite network showing connections between enriched terms (coloured rectangular nodes) and the individual genes (small circular nodes) that contributed to each term's enrichment. It is particularly useful for:

  • Identifying "hub" genes that contribute to multiple enriched terms simultaneously.
  • Spotting clusters of terms that are driven by the same subset of genes.
  • Distinguishing broad functional enrichment from enrichment driven by a small number of highly connected genes.

Node size for gene circles scales with degree (number of connected terms). You can drag nodes, scroll to zoom, toggle gene labels, and download the full network as a TSV.

Each results panel offers several download options:

  • Download TSV: Downloads all significant terms for the current category as a tab-separated file, including term label, name, hit count, raw p-value, adjusted p-value, enrichment direction, and the list of contributing gene models.
  • Download All PNGs: Exports all visible charts as PNG images.
  • SVG download: Available for the bar chart and dot plot tabs for publication-quality vector graphics.
  • CNET TSV: Downloads the node–edge table of the concept network for use in external network tools such as Cytoscape.

A few common reasons for empty results:

  • Gene list too small: Very few genes reduces statistical power. Try relaxing your upstream filtering criteria.
  • Background too narrow: If your custom background is small, many terms may not have enough genes to test. Consider using the full genome background.
  • P-value threshold too strict: Try raising the threshold from 0.05 to 0.1, or switching from Bonferroni to BH correction.
  • Wrong genotype: Make sure the gene model IDs match the selected page. Use the Load Genes button to verify detection.
  • Sparse annotation: Some NAM lines have less complete functional annotation than B73. If possible, try the B73 Enrichment page with orthologous B73 gene IDs.

Technical

FETA does not persistently store gene lists or results. All computation is performed server-side at the time of submission and results are returned directly to your browser session. No gene list data is retained after the session ends or shared with third parties.

FETA is built on a Python/Flask backend and can be run locally if you have access to the annotation data files. The source code and data file format specifications are available on request. There is currently no public REST API, but batch analysis via the command-line Flask routes is possible for users hosting their own instance.

FETA is tested and supported on current versions of Chrome, Firefox, and Safari. The interactive CNET network and cell diagram visualisations require JavaScript to be enabled. Internet Explorer is not supported.

Please contact the development team via email or open an issue on the project repository. When reporting a bug, it helps to include your browser version, the gene list you used, and a screenshot of the unexpected behaviour.
